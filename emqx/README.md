# EMQX configuration notes

EMQX is the secure MQTT entry point for the edge devices (Environment 1).

## Topic convention
Devices publish JSON telemetry to:

```
telemetry/<external_id>
```

Payload:

```json
{
  "device_id": "sensor_001",
  "timestamp": "2026-06-19T14:30:00Z",
  "metrics": { "temperature": 7.2, "humidity": 61.0 }
}
```

## TLS
The broker terminates MQTT over TLS on port **8883** using the certificates
generated by `ssl/generate-certs.sh` and mounted read-only at
`/opt/emqx/etc/certs`. Ship `ca.crt`, `client.crt` and `client.key` to each
device. `VERIFY=verify_peer` is enabled; set `FAIL_IF_NO_PEER_CERT=true` in
`docker-compose.yml` to enforce mutual TLS once every device has a certificate.

## Bridging to Kafka
Open-source EMQX has no native Kafka sink, so the lightweight
`mqtt-kafka-bridge` service subscribes to `telemetry/#` and republishes
normalised per-metric records onto the Kafka `raw-telemetry` topic. If you run
EMQX Enterprise you can replace the bridge with a built-in Kafka data
integration rule.

## Dashboard
Management UI: `http://<jetson-ip>:18083` (credentials from `.env`).
